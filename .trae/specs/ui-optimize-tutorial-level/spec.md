# UI 优化与教学关卡实现 Spec

## Why

当前项目 UI 框架功能完整（HUD、设置、主菜单），但存在以下问题：
1. **信息呈现不足** — HUD 缺少小地图、距离指示器、物品拾取反馈等关键游戏信息
2. **新手引导缺失** — 玩家首次进入无任何操作指引，学习成本高
3. **状态反馈薄弱** — 缺少死亡/撤离结果界面、暂停菜单等完整游戏流程 UI
4. **视觉体验朴素** — 纯色块+文字标签，缺少游戏氛围和视觉层次

参照《逃离鸭科夫核心机制与玩法设计方案》及官方 Wiki 的新手指南，需要构建完整的 UI 体系和第一个教学关卡。

## What Changes

### 一、UI 优化（6 个组件增强/新建）

#### 1.1 HUD 增强 (`game_hud.tscn` / `game_hud.gd`)
- **新增 Minimap 小地图**：右上角圆形小地图，显示玩家位置、敌人红点、撤离点绿标、边界范围
- **新增 DistanceIndicator 距离指示器**：屏幕右下角，显示武器有效射程状态（绿色=有效射程内，红色=超出）
- **新增 PickupNotification 物品拾取通知**：屏幕中央偏上浮动通知，显示拾取物品图标+名称+数量，3秒渐隐
- **新增 DamageNumber 伤害数字**：受击位置弹出飘字，伤害值颜色区分（白色=普通，红色=暴击，绿色=治疗）
- **重构信息面板布局**：左上角改为横向紧凑布局（血条+弹药一行，负重+金钱一行），增加图标前缀

#### 1.2 PauseMenu 暂停菜单 (新建)
- 全屏半透明遮罩 + 居中面板
- 按钮：继续 / 设置 / 返回主菜单 / 退出游戏
- ESC 键打开/关闭，自动暂停游戏

#### 1.3 ResultScreen 结果界面 (新建)
- **死亡界面**：显示"你已阵亡"、丢失物品列表、"回收尸体"/"返回基地"选项
- **撤离成功界面**：显示"撤离成功"、本次战利品统计（物品清单+总价值）、"返回基地"按钮

#### 1.4 TutorialOverlay 教学覆盖层 (新建)
- 高亮目标区域（半透明遮罩+发光边框）
- 文字提示框（标题+描述+按键提示）
- 步骤进度指示器（圆点进度条）
- "跳过教程"按钮（右下角）
- 支持键盘/鼠标继续下一步

### 二、教学关卡实现（5 阶段线性教学）

#### 2.1 TutorialZone 教学关卡场景 (新建 `tutorial_zone.tscn` / `tutorial_zone.gd`)
继承 CombatZone，增加教学状态机和阶段控制逻辑。

**地图设计**：训练场（Training Ground），约 1600×1200 像素的封闭区域

| 区域 | 坐标范围 | 功能 |
|------|---------|------|
| 出生区 | (100,100)-(300,300) | 玩家出生点，安全区域 |
| 移动训练区 | (350,100)-(700,300) | 开放空间，引导 WASD 移动 |
| 靶场区 | (750,100)-(1100,300) | 固定靶标，练习瞄准攻击 |
| 搜刮区 | (350,400)-(700,600) | 放置战利品容器，练习 E 交互 |
| 战斗训练区 | (750,400)-(1100,600) | 生成 1 个弱敌人，练习战斗 |
| 撤离区 | (1200,500)-(1500,700) | 撤离点，完成最终步骤 |

#### 2.2 教学阶段流程

```
阶段0: 欢迎 → 阶段1: 移动 → 阶段2: 攻击 → 阶段3: 搜刮 → 阶段4: 战斗 → 阶段5: 撤离 → 完成
```

| 阶段 | 名称 | 教学内容 | 完成条件 | 提示文本 |
|------|------|---------|---------|---------|
| 0 | 欢迎 | 欢迎来到鸭科夫训练场 | 按任意键/点击继续 | "欢迎来到鸭科夫！这里是训练场，将教你生存的基本技能。" |
| 1 | 移动 | WASD 移动角色 | 移动距离 > 200 像素 | "使用 W A S D 键移动你的角色\n到达标记区域以继续" |
| 2 | 攻击 | 鼠标瞄准 + 左键射击 | 击中靶标 3 次 | "移动鼠标瞄准目标\n按左键点击进行攻击\n击中靶标 3 次以继续" |
| 3 | 搜刮 | E 键交互拾取物品 | 拾取 1 个物品 | "走近战利品箱\n按 E 键搜刮物资\n这是获取装备的主要方式" |
| 4 | 战斗 | 战斗 + 翻滚闪避 | 击败 1 个敌人 | "敌人会发起攻击\n注意红色警告并翻滚(空格)闪避\n击败敌人以继续" |
| 5 | 撤离 | 到达撤离点并完成撤离 | 撤离进度 100% | "到达绿色烟雾标记的撤离点\n按住 E 键开始撤离\n成功撤离保住所有战利品" |

#### 2.3 GameManager 集成
- 新增 `go_to_tutorial()` 方法
- 新增 `GameState.TUTORIAL` 状态（可选，复用 PLAYING 亦可）
- 主菜单新增"教学模式"按钮入口

### 三、Player 增强（如需）

- **翻滚动作**（教学阶段4需要）：空格键触发无敌帧翻滚，冷却时间 1 秒
- 仅在教学阶段4及之后启用

## ADDED Requirements

### Requirement: 增强型游戏 HUD
系统 SHALL 提供包含以下信息的增强型 HUD：
- 左上角：生命值进度条（带数值）+ 弹药信息
- 左上角第二行：负重信息（超重红色警告）+ 金钱显示
- 右上角：圆形小地图（显示玩家、敌人、撤离点）
- 右下角：距离指示器（有效射程状态）
- 中央偏上：物品拾取浮动通知
- 底部中央：交互提示 + 撤离进度条

#### Scenario: 显示拾取通知
- **WHEN** 玩家拾取物品时 GameEvents.emit_item_picked_up 被调用
- **THEN** HUD 中央显示浮动通知（物品名+数量），3秒后渐隐消失

#### Scenario: 更新小地图
- **WHEN** 玩家/敌人/撤离点位置变化时
- **THEN** 小地图实时更新对应标记位置

### Requirement: 暂停菜单
系统 SHALL 提供功能完整的暂停菜单：
- ESC 键打开/关闭
- 包含：继续、设置、返回主菜单、退出游戏 四个选项
- 打开时暂停游戏时钟

#### Scenario: 打开暂停菜单
- **WHEN** 游戏处于 PLAYING 状态且按下 ESC 键
- **THEN** 显示暂停菜单面板，游戏暂停

### Requirement: 游戏结果界面
系统 SHALL 提供死亡和撤离两种结果界面：

#### Scenario: 显示死亡界面
- **WHEN** 玩家死亡时
- **THEN** 显示死亡界面，包含丢失物品列表和"回收尸体"/"返回基地"选项

#### Scenario: 显示撤离成功界面
- **WHEN** 玩家成功撤离时
- **THEN** 显示撤离成功界面，包含战利品统计和"返回基地"按钮

### Requirement: 教学覆盖层组件
系统 SHALL 提供 TutorialOverlay 组件用于引导新手：
- 支持高亮指定区域（Node2D 路径）
- 显示文字说明和按键提示
- 显示当前步骤进度
- 支持"跳过教程"功能
- 支持键盘/鼠标/触摸继续下一步

#### Scenario: 执行教学步骤
- **WHEN** 教学系统激活某一步骤时
- **THEN** 高亮对应区域，显示提示文字，更新进度指示器

### Requirement: 教学关卡
系统 SHALL 提供完整的教学关卡（TutorialZone）：
- 包含 6 个教学阶段（欢迎→移动→攻击→搜刮→战斗→撤离）
- 每个阶段有明确的完成条件
- 线性推进，不可跳过阶段（可整体跳过）
- 完成后解锁正常游戏模式

#### Scenario: 完成移动教学
- **WHEN** 玩家在阶段1累计移动超过 200 像素
- **THEN** 自动进入阶段2（攻击教学）

#### Scenario: 完成全流程教学
- **WHEN** 玩家完成阶段5（撤离）后
- **THEN** 显示"训练完成"提示，提供"前往基地"选项

### Requirement: 主菜单教学模式入口
系统 SHALL 在主菜单提供教学模式入口：
- 新增"教学模式"按钮
- 点击后进入 TutorialZone
- 首次游玩时可高亮推荐

## MODIFIED Requirements

### Requirement: 现有 HUD 布局
修改后的 HUD 采用分层布局：
- 信息面板从左上角垂直堆叠改为更紧凑的双行布局
- 新增 4 个独立功能模块（Minimap、DistanceIndicator、PickupNotification、DamageNumber）
- 保持现有信号连接方式不变

### Requirement: GameManager 状态管理
新增以下能力：
- `go_to_tutorial()` 场景切换方法
- 教学完成状态记录（PlayerData 中新增 tutorial_completed 标志）

### Requirement: Player 角色能力
新增翻滚动作（可选增强）：
- 输入映射：`roll` = 空格键
- 无敌帧持续时间：0.3 秒
- 冷却时间：1.0 秒
- 翻滚方向：当前移动方向或面向方向

## REMOVED Requirements

无。本规格为纯增量开发，不删除任何现有功能。
